## 들어가며

우테코 프리코스 3주차 과제 중 `Java의 Enum`을 적용해라는 추가 요구사항이 있었다. 평소에 Enum에 대한 정의만 뜬구름처럼 알고 있었는데 이 기회에 정리를 하면서 왜 Enum을 사용하는지에 대한 고찰도 가져보려고 한다.

## Enum이란?

상수들을 열거해주는 타입으로 `열거형` 이라고도 한다. 다른 언어들도 마찬가지로 Enum을 가지고 있는데 자바에서는 값뿐만 아니라 타입까지 체크해줌으로써 더욱 안전하다.

## Enum의 사용법

```java
enum 열거형 이름 { 상수명1, 상수명2, ... }
```

- Enum의 값을 가져오기 위해서는 클래스의 static변수처럼 `열거형이름.상수명` 으로 사용한다.
- Enum 상수간의 비교에는 `==` 를 사용할 수 있다.
- Enum의 비교는 `compareTo()` 이다.
    - 비교대상이 같으면 0, 왼쪽이 크면 양수, 오른쪽이 크면 음수를 반환

## Enum 내부 메서드

| 메서드 | 설명 |
| --- | --- |
| Class&#60;E&#62; getDeclaringClass() | Enum의 Class 객체를 반환 |
| String name() | Enum 상수의 이름을 문자열로 반환 |
| int ordinal() | Enum 상수가 정의된 순서를 반환 (0부터 시작) |
| T valueOf(Class&#60;T&#62; enumType, String name) | 지정된 Enum에서 name과 일치하는 Enum 상수를 반환 |

## Enum을 사용하는 이유(정리)

Enum을 사용하는 이유는 뭘까?라는 의문을 가졌다. 일단 관련된 상수들을 Enum 클래스로 한번에 관리할 수 있다는 장점이 있다. 
또한 타입의 안정성도 보장할 수 있다. 그런 예외적인 상황들을 덜 고려해도 된다는 점이 Enum을 사용하라고 하는 이유인거 같다.
앞으로 Enum을 적극적으로 적용해봐야겠다는 생각이 든다.
