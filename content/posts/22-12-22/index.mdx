### 들어가며

자바 컨벤션이나 Rest API 작성시 컨벤션을 작성하면서 문득 깃 레포지토리 네이밍 컨벤션이 궁금해졌다. 따로 git 페이지에선 설명해놓진 않았지만 예시로는 `hello-world`처럼 되어있었다. 레포지토리를 만들일이 많은데 대중적인 컨벤션을 따라 정하면 더 네이밍이 쉬울 것 같았다.

### stack overflow에서 찾은 naming convention에 대한 고찰

[Is there a naming convention for git repositories?](https://stackoverflow.com/questions/11947587/is-there-a-naming-convention-for-git-repositories)

스택 오버플로우에서 가장 핫(?) 했던 깃레포 네이밍 컨벤션이다.

### 그래서 어떻게?

1. 소문자로
2. dash `-` 로 구분
3. 최대한 상세하게 자세하게
